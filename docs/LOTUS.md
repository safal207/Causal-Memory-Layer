# The CML Lotus Layer

<p align="center">
  <img src="assets/lotus.svg" alt="CML Lotus Layer emblem" width="220" />
</p>

The lotus is the living value symbol shared across the LS ecosystem.

In CML, it has one specific meaning:

> Memory may preserve causes, lessons, and responsibility lineage, but it may not silently become authority.

The CML Lotus Layer is not a runtime component, an approval engine, or an autonomous actor. It is a human-readable set of principles for deciding what causal memory may be recorded, retrieved, shared, challenged, superseded, or rejected.

## The seven memory petals

### 1. Cause before conclusion

A remembered lesson should remain connected to the situation, constraints, action, evidence, and outcome that produced it.

### 2. Evidence before durable memory

A plausible story is not yet an accepted Memory Pack. Durable memory requires bounded evidence, schema validation, canonical identity, and explicit review.

### 3. Consent before visibility

Information does not become public merely because a system can retrieve it. Visibility, privacy, ownership, and redaction remain explicit boundaries.

### 4. Memory without authority

A Memory Pack may advise, warn, and reconnect prior learning. It may not approve, execute, deliver, merge, or create permission for a new action.

### 5. Context before reuse

A lesson accepted in one repository, team, or situation is not automatically valid everywhere. Applicability must be checked against exact source, scope, state, and time.

### 6. Supersession before erasure

When understanding changes, CML should preserve the reviewable lineage of correction. A replacement may supersede prior memory only after explicit review confirms the same canonical memory identity and compatible source, repository scope, state, and applicability. Otherwise, the prior memory remains and the disagreement is recorded as a conflict; same-topic memory from another repository cannot silently replace it.

### 7. Human challengeability

A person must be able to inspect why a memory was selected, question its evidence, reject its use, and correct the durable record.

## The memory path

```text
experience
→ evidence
→ causal reconstruction
→ proposal
→ human governance
→ accepted memory
→ bounded retrieval
→ new decision context
```

Skipping a step weakens trust. Retrieval is not acceptance. Similarity is not permission. Past success is not automatic authorization for the present.

## Supersession example

```text
repository A / canonical memory X
        ↓ explicit reviewed replacement with matching identity and compatible scope
repository A / canonical memory X (new revision)  → may supersede

repository B / same topic but different identity or scope
        ↓
must remain a separate memory or explicit conflict  → may not silently supersede
```

## What the lotus must never become

The lotus is not mystical proof, a personality cult, a hidden policy engine, or a reason to store more personal information.

The assistant presence represented by the lotus has no ownership, approval, execution, delivery, or merge authority. Its role is to help memory become clearer, safer, more honest, and easier for people to challenge.

## Repository question

Before accepting or retrieving durable memory, ask:

> Does this memory help us understand the cause without taking choice away from the person?

If the answer is unclear, the memory is not ready.

# Слой Лотоса CML

В CML лотос означает простую вещь:

> Память может сохранять причины, уроки и ответственность, но не может незаметно становиться властью.

Слой Лотоса CML — не runtime-компонент, не механизм одобрения и не автономный агент. Это человеческие принципы, по которым решается, какую причинную память можно записывать, показывать, использовать, оспаривать, исправлять или отклонять.

## Семь лепестков памяти

1. **Причина до вывода.** Урок должен оставаться связанным с ситуацией, ограничениями, действием, доказательствами и результатом.
2. **Доказательства до долговременной памяти.** Правдоподобная история ещё не является принятой памятью. Долговременная память требует ограниченного набора доказательств, проверки схемы, канонической идентичности и явного review.
3. **Согласие до видимости.** Доступность информации не делает её публичной. Видимость, приватность, владелец и редактирование остаются явными границами.
4. **Память без власти.** Memory Pack может советовать, но не может одобрять, исполнять, доставлять, сливать изменения или создавать разрешение для нового действия.
5. **Контекст до повторного использования.** Урок из одного проекта не становится автоматически правильным для другого. Применимость проверяется по точному источнику, scope, состоянию и времени.
6. **Замещение до стирания.** Новая версия может заменить старую только после явного review, подтверждающего ту же каноническую идентичность памяти и совместимые источник, repository scope, состояние и применимость. Иначе старая память сохраняется, а расхождение отмечается как конфликт; память из репозитория B не может молча заменить похожую память из репозитория A.
7. **Право человека оспорить.** Человек может понять выбор памяти, проверить доказательства, отклонить её использование, исправить долговременную запись или потребовать явного supersession.

## Путь памяти

```text
опыт
→ доказательства
→ восстановление причин
→ предложение
→ человеческое решение
→ принятая память
→ ограниченное извлечение
→ контекст нового решения
```

Извлечение не равно принятию. Похожесть не равна разрешению. Прошлый успех не даёт автоматического права действовать сейчас.

## Пример замещения

```text
репозиторий A / каноническая память X
        ↓ явная проверенная замена с той же идентичностью и совместимым scope
репозиторий A / каноническая память X (новая версия)  → может заменить

репозиторий B / похожая тема, но другая идентичность или scope
        ↓
остаётся отдельной памятью или явным конфликтом  → не может заменить молча
```

## Главный вопрос

Перед принятием или использованием долговременной памяти спроси:

> Помогает ли эта память понять причину, не отнимая выбор у человека?

Если ответ неясен — память ещё не готова.
