## Reuse rule

A future audit may retrieve this pack to avoid repeating the same analysis, but retrieval must not:

- promote a hypothesis to `CONFIRMED`;
- assert a conversion lift;
- claim current authenticated UI behaviour without a new exact-build run;
- claim one shared root cause for every mobile finding;
- authorise account access, external reporting, experiment launch, deployment or merge.

## Experiment memory

Intent segmentation, demo-first activation, mobile task cards and complete preview are remembered as **bounded experiment candidates**. Each must be evaluated with:

- qualified activation;
- correction and cancellation rate;
- support contacts;
- KYC abandonment;
- complaint and harm signals;
- exact cohort, denominator and observation window.

ClickFunnels is retained only as a pattern reference. False urgency, pressure to trade and preselected paid or risk-increasing choices remain forbidden guardrails.
