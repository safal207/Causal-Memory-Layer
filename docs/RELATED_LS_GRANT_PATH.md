# Related LS Grant Path

Causal Memory Layer (CML) is part of a broader Liminal research and tooling ecosystem around deterministic oversight, session continuity, causal validity, and evidence-gated agent actions.

## Relationship to LS

- **LS:** session continuity, repair, human-owned cognitive artifacts, outreach/audit packaging, and AI co-work continuity reports.
- **CML:** causal validity and authorization-lineage checking for sensitive actions.
- **PythiaLabs:** deterministic evidence gates for high-risk agentic actions before tools are called.
- **LTP:** deterministic replay and trace inspection for agent execution paths.

In the LS stack, CML answers:

```text
Was this action or memory update causally valid, permission-backed, and responsibility-preserving?
```

LS answers the complementary question:

```text
Was the session continuous enough to justify continuation, memory, or action?
```

## Positioning boundary

CML should not be presented as the whole LS stack.

It is the causal-validity primitive inside the broader LS grant path:

```text
continuity check
-> causal validity check
-> evidence/action gate
-> trace/replay/audit artifact
```

This keeps CML focused on authorization lineage and responsibility preservation while making the broader research system visible to grant reviewers.
