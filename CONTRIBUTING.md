# Contributing

## How to Contribute
- Open an issue describing the bug, proposal, or improvement.
- Keep pull requests focused and small.
- Include tests or validation updates for behavioral changes.
- Update documentation when interfaces or commands change.

## Pull Request Checklist
- [ ] Code is formatted and lint-clean.
- [ ] Tests/validation commands pass locally.
- [ ] README or docs are updated when needed.
- [ ] No generated artifacts or secrets are committed.

## Communication
Use clear reproduction steps, expected behavior, and current behavior in issues.
